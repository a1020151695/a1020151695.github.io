哈希表

Posted on 2021-07-12

哈希函数

根据Key来计算存储位置的计算规则我们称之为哈希函数

假如我们有一组数据,某位工程师每年的收入情况

直接定址法

Hash（Key）= A*Key + B。
优点：简单、均匀
缺点：需要事先知道关键字的分布情况(判断其对应关系是否为线性)

除留余数法

Hash( key ) = key % p ( p <= m)

m为地址数

p为一个不大于m，但最接近或者等于m的质数

哈希冲突

哈希函数算出来的位置已有元素, 即为冲突

冲突处理

线性探查法

产生冲突，依次查看其后的下一个桶，如果发现空位置插入新元素

平方探测法(二次探测法)

相对比上面的i+1, 平方探测法为 i+(平方序列)

平方序列: 1, -1, 4, -4, 9, -9

线性探测容易聚集，二次探测聚集情况较线性探测要好

散列表长度是某个4k＋3形式的素数时，平方探测法就可以探查到整个散列表空间

载荷因子

散列表的载荷因子：a = 插入元素个数 / 散列表的长度

a是散列表装满程度的标志因子。对于开放地址法，载荷因子非常重要，应严格限制在 0.7~0.8 以下。超过 0.8 ，查表时的CPU缓存按照指数曲线上升。

Posted on 2021-07-11

概念

连通图

图中任意两个点有路连通的叫做连通图

否则为非连通图

要使n个顶点的无向图连接，至少需要(n-1)(n-2)/2+1条边

n=4时，(3x2)/2+1=4，即最少有4条边就可构造连通图

当边的数量小于4时，都是非连通图

例题

当n=7时，连通图最小边数为(6x5)/2+1 = 16

28>16故，G必为连通图，不符合题意，排除

当n=8时，连通图最小边数为(7x6)/2+1 = 22

28>22故，G必为连通图，不符合题意，排除

当n=9时，连通图最小边数为(8x7)/2+1 = 29

29>28故，G可为无向非连通图，符合题意，故选C
